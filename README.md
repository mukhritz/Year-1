## 🍂 Semester 1: The Core Fundamentals

### 🧮 Discrete Structure
This course explores the mathematical foundations of computer science. It shifts the focus from continuous mathematics like calculus to discrete, countable systems. By analyzing structures like sets and graphs, we learn how to map out complex computing problems abstractly. These logical frameworks directly form the underlying basis for designing algorithms, complex data structures, and database query logic.

* **Key Concepts:** Set theory, logic and proofs, graph theory, combinatorics, and boolean algebra.

<blockquote> 
📖 **My Reflection:**
At first, the highly abstract nature of mathematical proofs and set theory felt completely disconnected from actual coding. However, everything clicked once we moved into graph theory, and I realized this is exactly how network routing and social media algorithms map data connections. It was a challenging mindset shift, but it drastically sharpened my analytical thinking.
<br><br>
🎯 **What I Gained:**  
Enhanced logical deduction skills, a strong grasp of algorithmic proof solving, and the ability to evaluate computational relationships structurally[cite: 1].
<br><br>
💡 **Areas for Improvement:**  
I need to spend more time practicing graph theory matrices and network tree applications, as these are vital for advanced data engineering[cite: 1].
</blockquote>

### 🔌 Digital Logic
Looking under the hood of a computer to understand how physical electronic components process binary information into 0s and 1s. This subject covers the design and analysis of the physical circuits that dictate how microprocessors handle arithmetic operations. By understanding these logic circuits, students learn how abstract software commands translate into physical hardware movements inside the CPU.

* **Key Concepts:** Logic gates like AND OR NOT, Boolean algebra, combinational circuits, and sequential logic design.

<blockquote>
📖 **My Reflection:**  
Witnessing how mathematical boolean algebra expressions physically map onto logic gates to build actual circuits was incredibly eye opening. The lab sessions where we designed circuit simulations felt like solving highly intricate puzzles. It made me appreciate the incredible complexity hidden beneath a computer power button.
<br><br>
🎯 **What I Gained:**  
The ability to design, map, and simplify combinational and sequential digital circuits using simulation tools[cite: 1].
<br><br>
💡 **Areas for Improvement:**  
I occasionally struggled with minimizing highly complex Karnaugh Maps, so I need to practice logic simplification to minimize gate counts more effectively[cite: 1].
</blockquote>

### 🖥️ Computer Organization and Architecture
Bridging the gap between software and hardware. This subject explains how a computer executes high level program instructions at the microscopic machine level. It covers the inner execution cycles of the CPU, memory hierarchies, and caching techniques that engineers use to maximize compute speeds. Understanding these low level operations allows developers to write code that perfectly exploits the physical hardware layout.

* **Key Concepts:** CPU structure, memory hierarchy like cache and RAM, instruction set architectures, and assembly language.

<blockquote>
📖 **My Reflection:**  
Writing low level assembly language code was an incredibly humbling experience. It made me realize how much modern programming languages like Python abstract away from us, and gave me a massive appreciation for how carefully a processor manages memory addresses and registers to maximize computing speeds.
<br><br>
🎯 **What I Gained:**  
Deep comprehension of internal CPU processing cycles, register management, memory hierarchies, and low level code execution mechanics[cite: 1].
<br><br>
💡 **Areas for Improvement:**  
I need to review hardware pipelining architectures more closely to fully understand how data execution blockages occur in physical processors[cite: 1].
</blockquote>
